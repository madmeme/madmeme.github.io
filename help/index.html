ParaText Paragraphs

Paragraph Alignment

Alignment can be done using one of the following codes at the beginning of a pragraph.

  • <<: left-aligned
  • >>: right-aligned
  • <>: justified
  • ><: centered

Text and Narrator Paragraphs

Text Paragraphs can be defined using --- on a single line. Each element in one text paragragph has to be figured out in order for the next text paragraph to appear. Below each text paragraph there can be one narrator paragraph which is separated using +++ on a single line. Elements in the narrator paragraph also have to be figured out.

Paragraph Example

 >>Abc Abc Abc
 Abc Abc Abc
 +++
 <<Help Help Help
 ---
 ><Abc Abc Abc
 +++
 Help Help Help
 ---

ParaText Elements

Check Boxes and Radio Buttons

Check boxes and radio buttons require the user to make a choice. Check boxes are created using x and X (checked). Radio buttons are created using o and O (selected). For radio buttons several options are possible.

Syntax

<Text1>X{Deco1#Id1|Id2|Id3}
<Text2>x{Deco2#Id1|Id2|Id3}
<Text3>x{Deco3#Id1|Id2|Id3}
<Text1>O{Deco1#Group|Id1|Id2|Id3}
<Text2>o{Deco2#Group|Id1|Id2|Id3}
<Text3>O{Deco3#Group|Id1|Id2|Id3}

Highlight Button

A highlight button can be used for highlighting text elements. The + clears the id, such that previous elements with the same id do not get selected. The + can be used on any id, in any paratext element. Highlight buttons must not have a deco text.

Syntax

<Text1>!{#Id1+|Id2+|Id3+}
<Text1>?{#Id1|Id2|Id3}

Combination

A combination requires the user to select different combinations. A combination is created using *. Each combination consists of several input and select elements which are denoted using ! and ? following the *, e.g. *!?? creates one input followed by two select elements.

|Combo2a|Combo2b|Combo2c
